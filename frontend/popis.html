Co je Pipeline Monitor?

Pipeline Monitor je webová aplikace pro evidenci, spouštění a sledování datových pipeline nad datovými zdroji, včetně monitoringu běhů a upozornění na problémy.

Jde o simulaci orchestrace a monitoringu — projekt napodobuje reálné systémy jako Apache Airflow, Databricks Jobs nebo AWS Glue, ale neprovádí skutečné distribuované výpočty.

Python 3 + Flask SQLite Vanilla JS REST API SPA Frontend

Rozsah aplikace

✅ Aplikace dělá

  • Eviduje datasety (zdroje dat)
  • Eviduje pipeline a jejich verze
  • Spouští pipeline manuálně
  • Sleduje běhy (runs) a jejich kroky
  • Zobrazuje stav běhů v reálném čase
  • Spravuje pravidla alertů
  • Automaticky vytváří alerty při selhání
  • Poskytuje dashboard s přehledem

❌ Aplikace nedělá

  • Neprovádí skutečné distribuované výpočty
  • Není to Spark ani Airflow
  • Neřeší fyzickou orchestraci clusteru
  • Neumí skutečný cron scheduler
  • Nepracuje s reálnými big data soubory
  • Nemá autentizaci / uživatele

Jak funguje simulace ETL běhu

Po kliknutí na Run pipeline se spustí simulace v pozadí (Python thread):

Extractnačtení dat
Transformzpracování
Loaduložení výsledků
  • Každý krok trvá náhodně 0,3–0,8 sekundy
  • S pravděpodobností 30 % jeden krok selže → run skončí failed
  • Při selhání se automaticky vytvoří alert event pro všechna aktivní pravidla
  • Při úspěchu se zaznamená náhodný počet zpracovaných záznamů (5 000–200 000)
pending → running → success / failed

Architektura backendu

Aplikace používá vzor Blueprint + Service Layer:

routes/

Pouze HTTP — přijímají request, volají service, vrací JSON. Žádná business logika.

services/

Veškerá business logika — validace, databázové operace, pravidla.

db.py + SQLite

Jednoduchá SQLite databáze s cizími klíči a CHECK constraints. Soubor pipeline_monitor.db.

threading.Thread

Simulace ETL běhu probíhá v pozadí — server neblokuje response a kroky se aktualizují asynchronně.

Jak spustit
